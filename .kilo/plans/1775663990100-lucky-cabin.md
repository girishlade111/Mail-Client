## Implementation Plan

### 1. Rules Data Model

**Rule structure:**
```javascript
{
  id: string,
  name: string,
  enabled: boolean,
  priority: number,
  matchAll: boolean, // AND vs OR logic
  conditions: [
    {
      field: 'from' | 'to' | 'subject' | 'body' | 'attachment' | 'size' | 
            'date' | 'label' | 'category' | 'account' | 'read' | 'starred' | 'domain' | 'keywords',
      operator: 'contains' | 'equals' | 'startsWith' | 'endsWith' | 'greaterThan' | 'lessThan' | 'before' | 'after',
      value: string,
      not: boolean // negation
    }
  ],
  actions: [
    {
      type: 'label' | 'move' | 'archive' | 'delete' | 'markRead' | 'markImportant' | 
            'star' | 'snooze' | 'category' | 'flag' | 'forward',
      value: string // folder id, label id, etc.
    }
  ],
  createdAt: Date,
  lastUsed: Date | null
}
```

### 4. Rule Engine

**In MailContext.jsx:**
```javascript
{
  rules: Rule[],
  applyRules(emailId): AppliedActions
}
```

**Engine function:**
- Takes email, returns applied actions
- Respects rule priority order
- Stops at first match (configurable)
- Only runs on: email arrival, manual trigger

### 5. Condition Fields

| Field | Operators | Value Type |
|------|-----------|-----------|
| from | contains, equals, regex | email/name |
| to | contains, equals | email/name |
| subject | contains, equals, regex | text |
| body | contains, equals | text |
| hasAttachment | is, is not | boolean |
| size | greaterThan, lessThan | MB |
| date | before, after, within | date/days |
| label | is, is not | labelId |
| category | is, is not | categoryId |
| account | is | accountId |
| read | is, is not | boolean |
| starred | is, is not | boolean |
| priority | is | high/normal/low |
| sender domain | contains, equals | domain |

### 6. Action Types

| Action | Value Type |
